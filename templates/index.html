Data Entry: Security Implementation

Describe how you implement each of the following controls for a grounded assessment.

1) Authorization

Describe how your application controls who can access what. Include how roles are assigned, how access is reviewed, and how privileged accounts are managed.

2) Authentication

Describe how users and service accounts authenticate. Include details on MFA, password requirements, SSO, and how default credentials are handled.

3) Logging & Monitoring

Describe what your application logs, where logs are stored, how long they are kept, and how they are reviewed.

4) Certification & Compliance

Describe what security standards or certifications your application or organisation complies with, and how compliance is audited and maintained.

5) Application Patching

Describe your patch management process, including how quickly different severity patches are applied and how legacy systems are handled.

6) System Hardening

Describe how your systems are hardened. Include configuration baselines used, how unused services are managed, encryption in transit, and admin interface controls.

7) Session Management

Describe how user sessions are managed. Include session timeout rules, how tokens are protected, and what happens when a user logs out.
